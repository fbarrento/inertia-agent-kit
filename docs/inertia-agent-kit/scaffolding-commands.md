# Scaffolding Commands

Status: Phase 2 command spec.

Use this as the source of truth for Inertia Agent Kit's resource, page,
feature, component, and story scaffolding commands.

## Purpose

IAK scaffolding gives agents the Laravel-like frontend split before they write
implementation code. The commands create page adapters, resource-local
features, colocated stories, typed fixtures, and generated-type imports.

The default app shape is not Feature-Sliced Design and does not create global
behavior buckets. IAK must not generate top-level `queries`, `actions`,
`forms`, `hooks`, or `composables` folders. Behavior starts in
`features/<resource>` unless it is pure framework-free code for `lib` or a
deliberately promoted reusable component.

Generated Wayfinder route/action output is a separate generator-owned contract
under the configured generated paths. Resource scaffolds may import it, but
they must not create or populate global behavior folders.

## Command Surface

Canonical commands:

```bash
iak new resource <resource>
iak new page <resource> <action>
iak new feature <resource> <name>
iak new component ui/<name>
iak new component app/<name>
iak new story <scope> <name>
iak plan validate <plan-file>
iak plan apply <plan-file>
```

All scaffold commands accept:

```txt
--adapter react|vue|svelte
--config iak.config.json
--dry-run
--apply
--json
--force
```

Agent mode:

- `IAK_AGENT=1` defaults command output to JSON.
- `--json` always emits JSON and avoids prose.
- `--dry-run` is the default unless `--apply` is present.
- `--apply` writes files only after producing the same plan shape used by
  dry-run.
- `--force` may overwrite only files with a matching previous scaffold hash or
  files explicitly marked as generated. It must not silently overwrite
  user-owned implementation files.

Resource options:

```txt
--controller App\\Http\\Controllers\\VehicleController
--route-name vehicles
--singular vehicle
--only index,show,create,edit
--except show
--with-stories auto|always|never
--allow-missing-generated-types
```

`--allow-missing-generated-types` may create imports and aliases that reference
expected generated types, but it must not create handwritten fallback copies of
backend Data classes.

## Resource Controller Mapping

`iak new resource <resource>` mirrors Laravel resource controllers.

```php
Route::resource('vehicles', VehicleController::class);
```

Default page mapping:

| Laravel action | Frontend page | Generated by resource scaffold |
| --- | --- | --- |
| `VehicleController@index` | `pages/vehicles/index` | yes |
| `VehicleController@show` | `pages/vehicles/show` | yes |
| `VehicleController@create` | `pages/vehicles/create` | yes |
| `VehicleController@store` | none | no, use generated Wayfinder action |
| `VehicleController@edit` | `pages/vehicles/edit` | yes |
| `VehicleController@update` | none | no, use generated Wayfinder action |
| `VehicleController@destroy` | none | no, use generated Wayfinder action |

Rules:

- The route resource name controls the page folder when available.
- Controller names map to kebab-case folders when route metadata is not
  available: `OrganizationSettingsController` becomes
  `pages/organization-settings/*`.
- Page files are route adapters only. They receive typed Inertia props, select
  layout, and compose feature or app components.
- Mutating controller actions are represented by generated Wayfinder imports,
  not standalone page files and not handwritten route strings.
- Nested resources use kebab-case path segments from the route:
  `organizations.vehicles` becomes `pages/organizations/vehicles/*` unless the
  project config defines a flatter route-to-page map.
- Ambiguous singular names require `--singular`, for example
  `--singular status` for `statuses`.

## Generated Resource Tree

React default for:

```bash
iak new resource vehicles --adapter react --apply
```

```txt
resources/js/
  pages/
    vehicles/
      index.tsx
      show.tsx
      create.tsx
      edit.tsx
  features/
    vehicles/
      vehicle.types.ts
      vehicle.fixtures.ts
      vehicle-table.tsx
      vehicle-table.stories.tsx
      vehicle-filters.tsx
      vehicle-form.tsx
      vehicle-form.stories.tsx
      vehicle-empty-state.tsx
```

The scaffold must not generate:

```txt
resources/js/queries/
resources/js/actions/
resources/js/forms/
resources/js/hooks/
resources/js/composables/
```

The same topology applies to Vue and Svelte adapters; only renderer-owned file
extensions and templates change.

Vue resource tree:

```txt
resources/js/
  pages/vehicles/
    index.vue
    show.vue
    create.vue
    edit.vue
  features/vehicles/
    vehicle.types.ts
    vehicle.fixtures.ts
    vehicle-table.vue
    vehicle-table.stories.ts
    vehicle-filters.vue
    vehicle-form.vue
    vehicle-form.stories.ts
    vehicle-empty-state.vue
```

Svelte resource tree:

```txt
resources/js/
  pages/vehicles/
    index.svelte
    show.svelte
    create.svelte
    edit.svelte
  features/vehicles/
    vehicle.types.ts
    vehicle.fixtures.ts
    vehicle-table.svelte
    vehicle-table.stories.ts
    vehicle-filters.svelte
    vehicle-form.svelte
    vehicle-form.stories.ts
    vehicle-empty-state.svelte
```

Generated files are intentionally small:

- `pages/<resource>/<action>.*` imports page prop aliases, layout, and feature
  components.
- `features/<resource>/<singular>.types.ts` composes generated backend types.
- `features/<resource>/<singular>.fixtures.ts` exports typed story fixtures.
- Feature components own resource-specific workflow UI.
- Stories are colocated beside reusable or stateful feature components.

## Page Command

`iak new page <resource> <action>` creates one page and any minimal supporting
feature files needed for that action.

Examples:

```bash
iak new page vehicles index
iak new page vehicles edit --with-stories auto
```

Action defaults:

| Action | Page file | Feature files |
| --- | --- | --- |
| `index` | `pages/vehicles/index.*` | table, filters, empty state, fixtures, types |
| `show` | `pages/vehicles/show.*` | summary/detail feature when configured |
| `create` | `pages/vehicles/create.*` | form, form story, fixtures, types |
| `edit` | `pages/vehicles/edit.*` | form, form story, fixtures, types |

Pages must not define:

- backend-derived prop interfaces;
- table or form implementation details;
- local enum labels;
- date, money, or number formatting;
- domain-specific reusable components.

## Feature Command

`iak new feature <resource> <name>` creates resource-local UI and behavior.

Examples:

```bash
iak new feature vehicles vehicle-table --kind table --with-story always
iak new feature vehicles vehicle-filters --kind filters
iak new feature vehicles vehicle-form --kind form --with-story always
```

Feature options:

```txt
--kind table|filters|form|empty-state|panel|custom
--with-story auto|always|never
--with-hook
--with-fixture
```

Rules:

- Hooks or composables are generated only under the feature folder and only
  when `--with-hook` is requested or the adapter template needs real
  orchestration logic.
- A generated hook file must be named locally, for example
  `features/vehicles/use-vehicle-filters.ts`.
- If the behavior is pure and framework-free, the plan should recommend `lib`
  instead of creating a hook.
- If the component becomes generic across resources, the plan should recommend
  `iak new component app/<name>` instead.

## Component Command

`iak new component ui/<name>` creates a design-system primitive.

```bash
iak new component ui/button --with-story always
```

Generated shape:

```txt
resources/js/components/ui/button.tsx
resources/js/components/ui/button.stories.tsx
resources/js/components/ui/button.spec.json
```

Primitive rules:

- domain-free;
- finite typed public API;
- semantic `ds-` utilities only for design-system styling decisions;
- colocated story required;
- optional `.spec.json` when the project keeps catalog contracts.

`iak new component app/<name>` creates a reusable app component.

```bash
iak new component app/filter-bar --with-story always
```

Generated shape:

```txt
resources/js/components/app/filter-bar.tsx
resources/js/components/app/filter-bar.stories.tsx
```

App component rules:

- reusable across resources;
- may compose primitives;
- must not contain resource-specific names or backend assumptions;
- colocated story required.

## Story Command

`iak new story <scope> <name>` creates or repairs a colocated Storybook file.

Examples:

```bash
iak new story vehicles vehicle-table
iak new story app filter-bar
iak new story ui button
```

Scope mapping:

| Scope | Component path | Story path |
| --- | --- | --- |
| `ui` | `components/ui/<name>.*` | `components/ui/<name>.stories.*` |
| `app` | `components/app/<name>.*` | `components/app/<name>.stories.*` |
| `<resource>` | `features/<resource>/<name>.*` | `features/<resource>/<name>.stories.*` |

Required story behavior:

- Use typed fixtures from the feature or component fixture file.
- Do not inline arbitrary fake backend-shaped objects.
- Include `Default` plus applicable states from:
  `Empty`, `Loading`, `Error`, `Disabled`, and `WithValidationErrors`.
- Include Storybook metadata needed to derive a stable story id.
- Use backend-like copy in fixtures only for story/test purposes; production
  components receive copy through props.

Feature stories are generated by default for components that are reusable,
stateful, visually important, exported outside their file, or likely to be
edited by agents. Pages do not receive stories by default.

## Adapter Differences

The plan shape is renderer-neutral. File extensions and templates come from
the configured adapter.

| Concern | React | Vue | Svelte |
| --- | --- | --- | --- |
| Page file | `index.tsx` | `index.vue` | `index.svelte` |
| Feature component | `vehicle-table.tsx` | `vehicle-table.vue` | `vehicle-table.svelte` |
| Story file | `vehicle-table.stories.tsx` | `vehicle-table.stories.ts` | `vehicle-table.stories.ts` |
| Component API | typed props interface | `<script setup lang="ts">` props | typed exported props |
| Local behavior | `use-vehicle-filters.ts` | `use-vehicle-filters.ts` composable in feature folder | `use-vehicle-filters.ts` or store in feature folder |
| Layout selection | page-level Inertia layout export/property | adapter template uses configured layout convention | adapter template uses configured layout convention |
| Slots/children | `children`/render props where needed | slots | slots/snippets per configured Svelte version |

Adapter invariants:

- File names are kebab-case for stability across renderers.
- Generated symbols use PascalCase component/type names.
- Type, fixture, and Wayfinder imports use the same aliases across adapters.
- Stories are Storybook CSF where the renderer supports it.
- The adapter must respect project path config; the table above shows v1
  defaults.

## Generated Type Imports

Scaffolds must compose generated backend contracts. They must not create local
copies of Laravel Data classes, enums, form payloads, filters, routes, or
validation shapes.

Default feature type file:

```ts
import type { App } from '@/types/generated'

export type VehicleIndexPageProps =
  App.Data.Vehicles.VehicleIndexPageData

export type VehicleShowPageProps =
  App.Data.Vehicles.VehicleShowPageData

export type VehicleCreatePageProps =
  App.Data.Vehicles.VehicleCreatePageData

export type VehicleEditPageProps =
  App.Data.Vehicles.VehicleEditPageData

export type VehicleListItem =
  App.Data.Vehicles.VehicleListItemData

export type VehicleFormValues =
  App.Data.Vehicles.VehicleFormData

export type VehicleFilters =
  App.Data.Vehicles.VehicleFiltersData
```

Default Wayfinder imports for form features:

```ts
import { store, update, destroy } from '@/actions/generated/App/Http/Controllers/VehicleController'
import { index, show, create, edit } from '@/routes/generated/vehicles'
```

Page templates may import the aliases from the feature type file:

```ts
import type { VehicleIndexPageProps } from '@/features/vehicles/vehicle.types'
```

Vue and Svelte templates use the same type aliases in their script blocks. The
adapter owns the renderer-specific prop syntax; the type source stays the
same.

If generated symbols are missing, the plan reports `missingGeneratedTypes`.
`--apply` should fail in strict mode. With `--allow-missing-generated-types`,
IAK may still write the aliases above, but no fallback frontend domain types.

## Backend-Owned Formatting And Copy

Scaffolds assume Laravel owns display text, translation, and locale-sensitive
formatting.

Generated production code should:

- render `props.copy.*`, field labels, placeholders, help text, and validation
  messages from backend Data props;
- render formatted date, money, number, enum, and status labels from
  display/value DTOs;
- use raw values only for form values, route parameters, comparisons, sorting,
  filtering, and request payloads;
- use generated Wayfinder helpers for actions and routes.

Generated production code must not include:

- `new Intl.NumberFormat(...)` or `new Intl.DateTimeFormat(...)` in render
  paths;
- date formatting library calls for user-facing text;
- local enum label maps;
- hard-coded validation messages;
- frontend translation dictionaries as the default architecture;
- handwritten Laravel route URLs or controller action contracts.

Story fixtures may contain realistic copy for review states, but stories must
keep that copy in typed fixtures and must not imply that production components
own translation.

## Dry-Run And Apply

Dry-run is the planning contract.

```bash
iak new resource vehicles --json
```

returns an `iak.scaffold-plan.v1` object and writes nothing.

Apply uses the same plan:

```bash
iak new resource vehicles --apply --json
```

Apply behavior:

- Validate `iak.config.json` and adapter availability.
- Resolve resource, singular, controller, route name, and page actions.
- Detect existing files and mark each as `create`, `skip`, `merge`, or
  `conflict`.
- Refuse conflicts unless `--force` is allowed by file ownership metadata.
- Write files atomically.
- Write or update scaffold metadata under `.iak/scaffolds/` when configured.
- Return the final plan with `status`, `writtenFiles`, `skippedFiles`, and
  `conflicts`.
- Recommend follow-up checks: type generation, typecheck, `iak audit --json`,
  Storybook tests, and `iak verify --json`.

`iak plan apply <plan-file>` must reject plans whose config hash, adapter id,
or target paths no longer match the current project unless `--replan` is used.

## Completion Gates

After applying a scaffold, agents should run the configured project checks:

```bash
iak audit --json
iak verify --json
```

When backend-generated contracts changed or were missing, run the configured
Spatie TypeScript transformer and Wayfinder generation commands before
typecheck. A scaffolded UI handoff is incomplete without audit, relevant story
or browser verification, and evidence paths in the final JSON handoff.
